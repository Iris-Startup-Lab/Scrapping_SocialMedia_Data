## 1. Sistema de Login (Usuario + Contraseña)

### Opción A: Supabase Auth (recomendada)

**Ventajas**: Ya tienes Supabase. Auth viene incluido. RLS se activa con `auth.uid()` real.

**Implementación**:

1. Activar Supabase Auth en Dashboard → Authentication → Providers → Email

2. Agregar página de login en Gradio:
```python
# ui/auth.py
def auth_page():
    with gr.Column():
        email = gr.Textbox(label="Email")
        password = gr.Textbox(label="Password", type="password")
        login_btn = gr.Button("Ingresar")
        signup_btn = gr.Button("Registrarse")
        status = gr.Markdown()
        return email, password, login_btn, signup_btn, status

def handle_login(email, password):
    from supabase import create_client
    client = create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
    try:
        resp = client.auth.sign_in_with_password({"email": email, "password": password})
        return resp.session.access_token, resp.user.id
    except Exception as e:
        return None, str(e)
```

3. Pasar `user_id` real (UUID de Supabase) en vez de `"00000000-..."`
```python
state.user_id = user.id  # UUID real de auth.users
```

4. Activar RLS para producción:
```sql
ALTER TABLE chismesito_gpt.unified_comments ENABLE ROW LEVEL SECURITY;
CREATE POLICY "user_own_data" ON chismesito_gpt.unified_comments
    FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
```

5. El `SUPABASE_SERVICE_KEY` en backend para inserts, `SUPABASE_ANON_KEY` para login en frontend.

**Esfuerzo**: 2-3 horas.

## 2. Calculadora de Tokens y Costo APIFY

### 2a. Contador de tokens (Gemini / Claude / DeepSeek)

Cada llamada al LLM consume tokens. Necesitas medir:

| Momento | ¿Qué se mide? |
|---------|---------------|
| Generar categorías | Tokens in (muestra 20%) + tokens out (lista Python) |
| Chat RAG | Tokens in (contexto + prompt) + tokens out (respuesta) |

**Implementación en `llm_manager.py`**:

```python
import tiktoken  # pip install tiktoken

TOKEN_USAGE = {"input": 0, "output": 0, "calls": 0}

def count_tokens(text: str, model: str = "gpt-4") -> int:
    """Estima tokens con tiktoken (funciona para OpenAI/DeepSeek)."""
    try:
        enc = tiktoken.encoding_for_model(model)
        return len(enc.encode(text))
    except:
        return len(text) // 4  # Fallback: ~4 chars por token

def get_llm_response_with_meter(prompt, model, system_prompt=None):
    t_in = count_tokens(prompt) + count_tokens(system_prompt or "")
    TOKEN_USAGE["input"] += t_in
    
    response = get_llm_response(prompt, model, system_prompt)
    
    t_out = count_tokens(response)
    TOKEN_USAGE["output"] += t_out
    TOKEN_USAGE["calls"] += 1
    return response
```

**Mostrar en UI** — agrega un componente en Gradio:


## 3. Mejorar Zero-Shot Classification

### Problema actual

BART-large-mnli es lento (~2s por comentario en CPU) y moderadamente preciso. Para 20 comentarios = 40 segundos.

### Opción A: Modelo más rápido (recomendado)

Cambiar a **DeBERTa-v3-base-mnli** (más rápido y preciso):

```python
# tools/zero_shot_tool.py
_classifier = None

def _get_classifier():
    global _classifier
    if _classifier is None:
        from transformers import pipeline
        _classifier = pipeline(
            "zero-shot-classification",
            model="MoritzLaurer/DeBERTa-v3-base-mnli-fever-anli",  # ~40% más rápido
            device=-1  # CPU
        )
    return _classifier
```

**Comparativa (CPU, ~100 tokens)**:

| Modelo | Tiempo/comentario | Precisión | Tamaño |
|--------|------------------|-----------|--------|
| BART-large-mnli | ~2.0s | 88% | 1.6 GB |
| DeBERTa-v3-base (multilingual) | ~1.2s | 91% | 700 MB |
| DeBERTa-v3-xsmall | ~0.3s | 85% | 140 MB |

**Recomendación**: `MoritzLaurer/DeBERTa-v3-base-mnli-fever-anli` si el texto está en inglés. Para español: `MoritzLaurer/mDeBERTa-v3-base-mnli-xnli` (multilingual).

### Opción B: Usar el LLM para clasificar (más lento pero más preciso)

En vez de zero-shot, pide al LLM que clasifique TODOS los comentarios de una vez:

```python
def classify_with_llm(comments: list[str], categories: list[str], model: str) -> list[str]:
    prompt = f"""Clasifica CADA uno de los siguientes comentarios en UNA de estas categorias:
{categories}

Por cada comentario, responde SOLO con la categoria. Un comentario por linea.

Comentarios:
""" + "\n".join(f"{i}. {c}" for i, c in enumerate(comments))

    response = get_llm_response(prompt=prompt, model=model)
    return [line.strip() for line in response.strip().split("\n") if line.strip()]
```

Ventaja: más preciso. Desventaja: usa tokens del LLM (costo), ~3s para lote de 20.

### Opción C: Batch processing

Procesar zero-shot en lotes con `pipeline(..., batch_size=8)`:

```python
_classifier = pipeline(
    "zero-shot-classification",
    model="MoritzLaurer/DeBERTa-v3-base-mnli-fever-anli",
    device=-1,
    batch_size=8  # Procesa 8 comentarios a la vez
)
results = _classifier(comments, candidate_labels, batch_size=8)
```

Reduce tiempo total ~60%.

### Recomendación final

| Prioridad | Cambio | Ahorro |
|-----------|--------|--------|
| Alta | DeBERTa-v3 en vez de BART | -40% tiempo |
| Media | Batch size = 8 | -60% tiempo adicional |
| Baja | LLM como clasificador | Solo si necesitas máxima precisión |
